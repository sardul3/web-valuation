Cycles

  • Cycles represent the container for a unit of evaluation data recorded during a specified period of time
    • For example, a cycle that was started in Spring 2016 and ended in Fall 2018 contains all of the data (rubric templates, rubric scores, outcomes, measures, evaluators, etc) that was recorded/created during the time the cycle was running
  • Cycle reports can be generated to enable the interpretation of data trends, outliers that may be the cause of lower average scores, and other statistics that could be the cause of a programs satifactory/unsatisfactory performance

  1. Find 'Start New Cycle' on the user dashboard
  2. In the year field input the year the cycle will start
  3. In the semester field input the semester the cycle will start
  4. Select 'Start'
Complete the input fields and select 'Start'

  • If there are multiple running cycles, follow the instrustions:

Outcomes

  • An outcome is the description for a specific criterion that a program must meet
    • Example: Students can communicate effectivley
  • An outcome is associated with various measures, which use tools (rubrics, test scores) to provide the physical evidence of a program's success

The creation of an outcome can occur in various ways:
  • It is possible to already be working in the desired cycle
    1. Select 'Outcome' on the navigation bar
    2. Select '+' by the 'Outcomes' heading
  • A cycle could have just been created and outcomes need to be created for it
    1. Select '+' by the 'Outcomes' heading
  • The cycle for the new outcome must be accessed before the outcome can be created
    1. Find 'Running Cycles' on the admin dashboard
    2. Select 'Visit' by the desired cycle
    3. Select '+' by the 'Outcomes' heading
After following the steps for one of the scenarios above:

Measures

  • A mechanism used to link a tool to an outcome in order to achieve proving the success/failure of an outcome
  • A statement of conditions that a specific type of tool will meet
  • Example: At least 75% of students will receive an average rubric score of 3 of greater when evaluated using the oral communication rubric
    • The measure sets a standard that a specific tool will meet
    • The measure passes if the standard is met, which will contribute to the success of the outcome

  1. Access the outcome the measure will be associated with
  2. The outcome a measure will be associated with must be created before the measure
    • Check out 'Access Outcome' under the Outcome heading
  3. Select '+' to add a measure
  4. Fill out the Measure Title field with the measure title
  5. Fill out the Cutoff-percent field with the cutoff-percent
    • The cutoff-percent is the percent of students that must meet the cutoff score
    • Example: At least 75% of students will receive an average rubric score
  6. Select the cuttoff-score type
    • Will the students be expected to achieve an average score, score in a minimum percentile, or score a minimum percent
    • Percentage Example: Students will answer correctly 60% or more of questions
    • Average Example: Students will have an average rubric score of 3 of greater
    • Percentile Example: Students will score at or above the 50th percentile
  7. Fill out the 'cutoff-score' field with the cutoff-score
    • The score students are expected to meet in order for the measure to pass
  8. Select the tool type
    • The type of tool that will be used to prove the measure
    • Rubrics can be weighted or unweighted
    • Test scores can be input individually or in batched by uploading a .csv in the correct format
  9. Select 'Add' to complete creation

Rubrics

  1. Select 'Rubrics' on the admin dashboard
  2. Select '+' to add a new rubric
  3. Enter the number of rows
    • The number of rows should includ a row for the grading values, and a row for each criterion
    • Example: 5 criterion, 1 + 5 = 6 rows
  4. Enter the number of columns
    • The number of columns should include a column for the critera list, and a column for each grading value
    • Example: rubric scale is 1-5, 1 + 5 = 6 columns. If using words such as poor, average, mastery, advanced; 4 words + 1 = 5
  5. Select Yes or No for weighted rubric
    • Select Yes: If the criterion will not be graded equally. The weights for each criterion will be applied during rubric creation
    • Select No: If the criterion will be graded equally
  6. Select whether the order will be ascending/descending
    • Select Ascending: If the rubric values will be labled worst to best (ie. Poor-Excellent, 1-5)
    • Select Descending: If the rubric will be scored best to worst (ie. Excellent-Poor, 5-1)
  7. Select 'Create'
Select 'Rubrics' on the navigation bar
Select '+' by 'Rubrics'
Complete the input fields and select 'Create'

    Proceed with the following steps after creating a rubric
  1. Fill out the first column with the grading categories
  2. Fill out the remaining cells of the rubric with the grading descriptions associated with the row criteria of the cell and the value associated with the column of the cell
  3. Select 'Add Rubric'
Fill out the first column with the grading categories
